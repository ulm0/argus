# Argus

[![Release](https://img.shields.io/github/v/release/ulm0/argus?style=for-the-badge)](https://github.com/ulm0/argus/releases/latest)

Edge-optimized dashcam manager for Tesla vehicles. Stores, manages, and processes Dashcam and Sentry Mode data on a Raspberry Pi Zero 2 W.

## Architecture

- **Backend:** Go -- all system operations (USB gadget, filesystem, WiFi, Samba, Telegram alerts) implemented natively
- **Frontend:** Next.js -- static export embedded into the Go binary
- **Delivery:** Single self-contained binary with zero runtime dependencies
- **Target:** Raspberry Pi Zero 2 W (minimal CPU/RAM footprint)

## Project Structure

```
cmd/argus/        Go entry point + CLI commands (embeds web frontend)
internal/         Go packages (API, services, system interfaces)
web/              Next.js frontend source
templates/        systemd service unit
config.yaml       Reference configuration (auto-generated by setup)
Makefile          Build orchestration
```

## CLI Commands

```
argus run [--config path]                   Start the web server
argus generate [--output path] [--force]   Generate initial config.yaml (edit then run `sudo argus setup`)
argus setup [--dir path] [--cam-size 50G]   Install on a Raspberry Pi (requires root)
argus upgrade [--yes]                       Upgrade to the latest release (requires root)
argus remove [--yes] [--keep-images]        Uninstall from the system (requires root)
argus version                               Print the build version
```

### Install

The binary is self-contained — no other files are required at download time. Run this one-liner on the target device and it will auto-detect the architecture, download the correct release, install the binary to `/usr/local/bin/`, and set up everything:

```bash
curl -fsSL https://github.com/ulm0/argus/releases/latest/download/argus_$(uname -s | tr '[:upper:]' '[:lower:]')_$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/;s/armv7l/armv7/;s/armv6l/armv6').tar.gz | sudo tar -xz -C /usr/local/bin argus

argus generate

sudo argus setup
```

Or download manually from the [releases page](https://github.com/ulm0/argus/releases/latest).

Or build from source:

```bash
make all  # builds frontend + cross-compiles binary
```

Options for `setup`:

| Flag | Default | Description |
|---|---|---|
| `--dir` | `~/.argus/` | Data directory |
| `--config` | `<dir>/config.yaml` | Config file path |
| `--cam-size` | `50G` (env: `ARGUS_CAM_SIZE`) | TeslaCam disk image size |
| `--show-size` | `10G` (env: `ARGUS_SHOW_SIZE`) | LightShow disk image size |
| `--music-size` | `32G` (env: `ARGUS_MUSIC_SIZE`) | Music disk image size |

### Uninstall

```bash
sudo argus remove
# or skip all prompts:
sudo argus remove --yes --keep-images
```

### Run manually

```bash
argus run
# or with explicit config:
argus run --config ~/.argus/config.yaml
```

## Build

```bash
# Install dependencies
make deps

# Build frontend + cross-compile Go binary (arm64/linux)
make all

# Build for local development
make build-local

# Build snapshot release locally (no tag needed)
make snapshot

# Publish a release (requires git tag + GITHUB_TOKEN)
make release

# Run tests
make test
```

The compiled binary lands in `bin/argus`.

## Development

```bash
# Frontend dev server (hot reload)
cd web && pnpm dev

# Backend
go run ./cmd/argus run config.yaml
```

## Configuration

All settings live in `~/.argus/config.yaml`, generated automatically by `argus setup`.

Key sections:

| Section | Purpose |
|---|---|
| `installation` | Target user (for Samba), mount directory |
| `disk_images` | Image names, labels, filesystem types |
| `network` | Samba password, web port |
| `offline_ap` | Hotspot fallback settings |
| `telegram` | Bot token, chat ID, alert behavior |
| `update` | Auto-update opt-in, startup check |
